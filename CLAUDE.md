# CLAUDE.md — partycalc

Project instructions for Claude Code working in this repo. Inherits the ElevatedProgress
venture playbook from the parent folder's CLAUDE.md.

## What this is

A zero-dependency static-site generator. `generate.js` reads `data/items.json` + `assets/`
and writes ~360 HTML pages into `public/`. It's an SEO-driven, AdSense-monetized party
quantity site: "how many pizzas / how much ice for N people" answer pages for 21 party
staples × 16 guest counts, plus interactive per-item calculators and a full menu planner.
Target: https://party.elevatedprogress.com/. Audience is general/US consumers — clean,
fast, light UI.

## The one architectural rule

**All quantity math lives in `assets/party-math.js`** — a tiny UMD module required by
`generate.js` at build time AND loaded by the browser tools. Never duplicate a portion
formula anywhere else; static pages and live calculators must always agree. `data/items.json`
holds only copy (names, blurbs, tips) and the guest-count list.

## Deploy — just push

`git push` to `main` is the deploy — GitHub Actions (`.github/workflows/deploy.yml`) runs
the generator and publishes to Pages.

- **Never manually build and commit output.** `public/` is git-ignored build output.
- **Never hand-edit anything in `public/`.**
- Commit as the neutral identity, NOT a personal name:
  `git -c user.name="partycalc" -c user.email="partycalc@users.noreply.github.com" commit …`

## Local build / preview

```
node generate.js     # writes ./public
node server.js       # preview at http://localhost:5057
```

## Adding an item

Add the copy entry in `data/items.json` (slug, name, q, emoji, blurb, tip) AND a matching
math function in `assets/party-math.js` keyed by the same slug. Portions should follow
standard catering rules — cite the rule in the `sub` line ("3 slices per person…").

## Don't break these (generated by generate.js, must keep serving)

- `ads.txt` + the AdSense loader in `<head>` — publisher `ca-pub-5580575158570188`.
- `sitemap.xml`, `robots.txt`, `.nojekyll`, `CNAME` (party.elevatedprogress.com).
- A Google Search Console verification file will be added once the property is verified.

## Config knobs

`DOMAIN` and `BASE`, same semantics as countdownkit. Production values live in the workflow.
